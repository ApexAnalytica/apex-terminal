01. System Overview

APEX Terminal is a causal intelligence platform for analyzing systemic risk in critical infrastructure supply chains. It combines interactive 3D graph visualization with four specialized causal reasoning engines.

04. 3D Rendering Pipeline

DATA TO PIXELS

    |
    v
computeLayout3D(nodes, edges, existingPositions?)
    |-- d3-force-3d simulation
    |   |-- Link force: distance=25, strength=0.4
    |   |-- Charge force: strength=-120 (repulsion)
    |   +-- Center force: origin (0,0,0)
    |
    |-- Iterations: 200 (cold start) or 50 (incremental/import)
    |-- Domain z-layering (10 target layers, shifted x0.7)
    +-- Normalize to bounds: +/-55 (x), +/-40 (y), +/-35 (z)
        |
        v
    NodePosition[] { id, x, y, z }
        |
        +----> DAGNode3D (per node)
        |       Size:  0.5 + (omega/10)^2.2 x 4.5  =  [0.5 - 5.0]
        |       Color: getCategoryColor(category) or grey if disconnected
        |       Glow:  red (omega > 9) / amber (>= 7) / green
        |       Label: HTML overlay at distance factor 35
        |       Hover: detail card with 4-axis omega bars
        |
        +----> DAGEdge3D (per edge)
        |       Shape: Quadratic Bezier tube geometry
        |       Color: directed=cyan / temporal=amber / confounded=orange
        |       Arrow: cone at 80% along path
        |       Hover: physical mechanism label
        |
        +----> CameraRig
                Home:      [40, 30, 80] looking at origin
                Selection: animate to [node + (18,14,30)] over 400ms
                Easing:    ease-out cubic

Disconnected Node Detection

After any graph change (import, scissors), the renderer computes connected components via BFS. Nodes outside the largest connected component render as grey. When new edges bridge a floating cluster to the main graph, those nodes transition to full color automatically.

05. Core Engines

APEX Terminal has four reasoning modules, each corresponding to a foundational research approach:

SPIRTES Structure Discovery

Causal structure discovery using constraint-based and score-based algorithms.

  • DCD (Directed Cyclic Discovery) — primary structure learning
  • PCMCI+ — temporal lag analysis for time-series causation
  • FCI — latent confounding detection under hidden variables

The Trinity Panel visualizes all three discovery methods as SVG circular layouts, allowing comparison of which edges each method identifies.

Cascade Analysis: Computes spectral radius (lambda_max) of the weighted adjacency matrix. If lambda_max < 1.0, the system is stable. Top-3 centrality nodes are identified by degree centrality. Displayed as: dS/dt = -damping*S + forgettingRate

TARSKI Truth Verification

Physical constraint validation using a formal axiom library.

Axiom Levels

LevelTypeCountExamples
L0Physics (immutable)6Temporal Priority, Conservation of Flow, DAG Integrity, Carnot Bound, Shannon Capacity
L1Regulatory (red alert)4Sanction Logic (OFAC/EU), Force Majeure, Capital Adequacy, EUV Export Control
L2Heuristic (anomaly flag)2Lead-Lag Reversal, Capacity Saturation

In VERIFIED mode, restricted nodes display a red badge and inconsistent edges render as red dashed lines. Proof traces show which axioms were violated, the solver used (Z3 or cvc5), and check duration.

PEARL Causal Reasoning

Pearl's do-calculus for counterfactual reasoning and graph surgery.

Intervention Mode (do(X))

When a target node is selected, the system computes:

  • Expected Utility: E[U] = target.omega x avg_downstream_weight x 10
  • Regret Bound: (1 - avg_weight) x target.omega
  • Affected Nodes: BFS downstream with delta-omega estimates
  • Method: DeepCFR (if confounded paths) or BackdoorAdjustment

Scissors Tool (Graph Surgery)

Cutting an edge triggers severEdgeAndSpawnConsequences():

  1. Edge marked as severed (red, dashed, 25% opacity)
  2. 2 consequence nodes spawned from domain-specific templates (omega 7.5–9.5)
  3. New edges connect consequences to the original edge's target (weight 0.8)
  4. Consequence nodes render orange with birth animation

PARETO Risk Assessment

Omega-Fragility scoring and doomsday clock for systemic risk monitoring.

06. Dataset Import Pipeline

Fully client-side import system supporting four file formats, with validation, preview, and merge-into-graph behavior.

Supported Formats

FormatExtensionsParser Method
JSON.jsonJSON.parse() — accepts { nodes, edges } or arrays
CSV.csvString splitting with quoted field handling, header aliasing
GraphML.graphml, .gmlDOMParser XML parsing, <key> mapping
DOT.dot, .gvRegex-based: digraph { node [attrs]; a -> b [attrs] }

Import Flow

FILE TO GRAPH
  File dropped/selected
      |
      v
  detectFormat(file)
      |-- Check extension (.csv, .json, .graphml, .dot)
      +-- Fallback: content sniffing (starts with '{', '<?xml', 'digraph')
      |
      v
  parseFile(file) -> ParsedGraph { nodes: RawNode[], edges: RawEdge[] }
      |
      v
  validateParsedGraph(parsed, existingGraph)
      |-- ERRORS (block import):
      |   |-- Duplicate node IDs within file
      |   +-- Edge source/target references non-existent node
      |-- WARNINGS (allow import):
      |   |-- Node/edge ID already in graph (will skip on merge)
      |   |-- Invalid category, edge type (coerced to defaults)
      |   +-- Numeric fields out of range (clamped)
      +-- Apply defaults: applyNodeDefaults(), applyEdgeDefaults()
      |
      v
  ValidationResult { valid, issues[], resolvedNodes[], resolvedEdges[] }
      |
      v  (user reviews preview table, clicks MERGE)
  mergeGraphs(existing, incoming)
      |-- Skip duplicate node/edge IDs
      |-- Skip edges with invalid refs
      |-- Recompute metadata (density, counts)
      +-- Set verificationStatus = "UNVERIFIED"
      |
      v
  mergeGraphData(nodes, edges) -> store update
      |-- graphData = merged
      |-- initialGraph = merged
      |-- Layout recomputes (50 iterations, preserves positions)
      |-- Disconnected nodes -> grey
      +-- Copilot: "Dataset imported: +X nodes, +Y edges"

Merge Behavior

Always merge, never replace. Imported data is added on top of the existing graph. Duplicate IDs are skipped. Disconnected imported clusters render grey until edges are added that bridge them to the main graph.
